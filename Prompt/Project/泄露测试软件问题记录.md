# 三、数据库表设计要求

如果项目中已经存在 `operation_log` 表，请以当前表结构为准，不要随意破坏已有字段。

如果字段不完整，请补充必要字段，并同步更新 SQL 文档。

建议 `operation_log` 表至少包含以下字段：

```sql
id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',

user_id BIGINT NULL COMMENT '操作用户ID',
username VARCHAR(100) NULL COMMENT '操作用户名',
role_name VARCHAR(100) NULL COMMENT '用户角色名称',

operation_type VARCHAR(100) NOT NULL COMMENT '操作类型，例如 test_start/config_save/alarm_ack/user_add',
operation_name VARCHAR(200) NOT NULL COMMENT '操作名称，例如 开始测试/保存系统配置/确认报警',
operation_module VARCHAR(100) NULL COMMENT '操作模块，例如 test/config/alarm/user/system',

operation_source VARCHAR(50) NULL COMMENT '操作来源，例如 qml/api/external_touch/system',
target_type VARCHAR(100) NULL COMMENT '操作对象类型，例如 test_record/system_config/alarm_record/sys_user',
target_id BIGINT NULL COMMENT '操作对象ID',

request_data TEXT NULL COMMENT '请求参数摘要',
before_data TEXT NULL COMMENT '操作前数据摘要',
after_data TEXT NULL COMMENT '操作后数据摘要',
result VARCHAR(50) NOT NULL COMMENT '操作结果 success/fail',
error_message TEXT NULL COMMENT '失败原因',

client_ip VARCHAR(100) NULL COMMENT '客户端IP',
device_name VARCHAR(100) NULL COMMENT '设备名称',
remark VARCHAR(255) NULL COMMENT '备注',

operation_time BIGINT NOT NULL COMMENT '操作时间，毫秒级Unix时间戳',
created_at BIGINT NOT NULL COMMENT '创建时间，毫秒级Unix时间戳'
```

要求：

1. 时间字段统一使用 `BIGINT`。
2. 时间戳使用毫秒级 Unix 时间戳。
3. `operation_type` 用于程序判断。
4. `operation_name` 用于页面显示。
5. `operation_source` 用于区分页面操作、外部接口操作、系统自动操作。
6. `request_data`、`before_data`、`after_data` 不要保存明文密码。
7. 失败操作也要记录。
8. 如果用户未登录，则 `user_id` 可以为空，但 `operation_source` 必须记录为 `system` 或 `external_touch`。

建议索引：

```sql
INDEX idx_operation_time (operation_time),
INDEX idx_user_id (user_id),
INDEX idx_operation_type (operation_type),
INDEX idx_operation_module (operation_module),
INDEX idx_operation_source (operation_source),
INDEX idx_target (target_type, target_id)
```

1. HTTP REST API 适合外部工控机主动调用，例如开始测试、停止测试、读取状态、读取配方、保存配置。
2. WebSocket 适合实时推送状态，例如测试进度、压力曲线、报警信息、设备状态。
3. 调试方便，可以用 curl、Postman、浏览器、Python 脚本测试。
4. 文档容易编写，其他工控机容易对接。
5. 适合 Qt / QML 应用作为本地服务程序运行。

如果当前 Qt 版本或项目依赖不适合直接使用 HTTP 服务，可以采用：

```text
QTcpServer + JSON Line Protocol
```

作为备选方案。

不要一开始就强行引入复杂协议。Modbus TCP、OPC UA、MQTT 只有在项目已有需求或已有依赖时再补充说明，不要盲目引入。
